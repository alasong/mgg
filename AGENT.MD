## 项目概述

### 项目目标
mgg 是一个 CLI 项目管理工具，通过派发独立的 Claude Code 子进程来执行开发任务，实现自动化开发工作流。

### 核心价值
- 将开发任务自动化派发到 Claude Code，减少人工重复操作
- 通过 Skill 路由系统将任务匹配到最适合的执行单元
- 提供决策通道，在自动化流程中保留人工决策节点
- 支持任务依赖编排，实现复杂工作流

### 技术栈
| 类别 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 语言 | Python | >=3.10 | 主开发语言 |

## 架构规范

### 分层架构
```
┌─────────────────────────────────────┐
│           CLI 层 (Presentation)      │
│  - argparse 命令解析                  │
│  - 用户交互 / 输出格式化              │
└─────────────────────────────────────┘
                    │
┌─────────────────────────────────────┐
│           调度层 (Orchestrator)       │
│  - Skill 路由 / 依赖等待              │
│  - 决策通道 / 任务编排                │
└─────────────────────────────────────┘
                    │
┌─────────────────────────────────────┐
│           执行层 (Execution)          │
│  - Claude Code 子进程管理             │
│  - 输出解析 / 状态追踪               │
└─────────────────────────────────────┘
                    │
┌─────────────────────────────────────┐
│           持久层 (Persistence)        │
│  - .mgg/tasks/ 任务状态 + 结果       │
│  - decision.json 决策记录            │
└─────────────────────────────────────┘
```

### 依赖规则
- **单向依赖**：CLI → 调度层 → 执行层 → 持久层
- **接口隔离**：层间通过函数签名通信
- **模块化**：每个核心功能独立函数，不耦合

### 模块边界
| 模块 | 职责 | 对外接口 | 依赖模块 |
|------|------|----------|----------|
| CLI (cmd_*) | 命令解析、参数校验 | `mgg run/status/ls/resume/decide` | 调度层 |
| Skill 路由 | 根据 prompt 匹配最优 skill | `_infer_skill()` | 执行层 |
| 决策通道 | 从输出提取决策、提示用户 | `_extract_decision_from_output()` | 持久层 |
| Claude 子进程 | 执行实际任务 | `subprocess.run(["claude", "-p", ...])` | 持久层 |
| 任务持久化 | 状态读写、决策保存 | `_load_task()` / `_save_task()` | 无 |

## 开发流程

遵循 ADE V2.0 规范的 PDCA 循环：

```
Spec (定义迭代做什么)
  → Skill (定义怎么做)
    → TDD (原子实现)
      → 执行状态文件 (追踪进展)
```

1. **Plan**: 创建/更新 Spec，定义迭代目标和范围
2. **Do**: Spec 引用 Skill → Skill 按 PDCA 执行 → TDD 原子实现
3. **Check**: 运行测试、审查决策通道、验证覆盖率
4. **Act**: 更新复盘、Skill 优化、AGENT.MD 更新
